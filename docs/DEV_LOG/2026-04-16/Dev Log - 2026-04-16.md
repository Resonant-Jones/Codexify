**Narrative Log**

Dev Log - 2026-04-16

This was a day about removing accidental ambiguity. The first problem showed up in Guardian’s front end: too many visual and runtime states were being inferred indirectly, so the UI could drift from the truth it was supposed to display. We answered that by adding a canonical runtime visual-state mapping layer, wiring in a provider-state feed, and then trimming back debug rendering once the state was explicit enough to stand on its own.

The composer work followed the same pattern. The lineage bug for synthetic threads, the placeholder overlap, and the send-button spacing all pointed to the same underlying issue: the composer was carrying too much incidental complexity. We polished the interaction contract, then backed off the heavier hierarchical styling when it proved noisier than the minimal inline version. That rollback mattered. It was less interesting cosmetically, but more truthful operationally. Along the way, we also captured the `Chronicle v1` spec so continuity stays a derived layer, not a shadow memory system.

The second half of the day moved into retrieval. `obsidian_only` is now an explicit mode with strict enforcement, and `personal_knowledge` now guarantees Obsidian instead of treating it as optional. That is a small wording change on paper and a real boundary change in the system. The broker, policy layer, route wiring, and tests now agree about what the mode means, which reduces the chance of silent scope drift when retrieval starts widening.

What this makes possible next is a cleaner live proof. The UI state is more legible, the composer has fewer ways to lie about itself, and the retrieval surface now describes its own boundaries more honestly. That is not release readiness by itself, but it is the kind of structural cleanup that makes the next verification pass smaller and more dependable.
