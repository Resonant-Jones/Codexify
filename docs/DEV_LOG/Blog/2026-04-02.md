# Daily Log — April 2, 2026

Today felt different. Not because everything suddenly became polished. Quite the opposite. The moment that changed things was finally seeing **Persona Studio** and **Command Center** as real rendered surfaces instead of abstract intentions. They were rough immediately. A little lopsided. A little underfed. Command Center in particular looked like it had been assembled from whatever was left in the workshop after the real furniture got picked over.

And that was good news.

Once the interfaces existed in front of me, the work stopped being speculative. I was no longer trying to improve a mental model. I was reacting to actual surfaces with actual problems. Hierarchy problems. legibility problems. state-truth problems. The kind of problems that can be solved directly.

So I split the work cleanly.

## Persona Studio

Persona Studio needed to stop feeling like a concept demo and start feeling like a real editing surface.

The first pass was structural. I strengthened the hierarchy so the center editor actually reads like the primary work area. The tabs became more legible. The layout rhythm improved. The diagnostics rail stopped competing so aggressively with the part of the page that is supposed to matter most.

That alone helped, but the bigger shift came later.

I replaced the fake edit behavior with a real local draft model. Saved persona state and draft state now live separately. Editing touches the draft, not the saved record. **Save** now actually persists. **Reset** actually restores. **Save As New** actually duplicates the draft into a new persona instead of pretending to. Dirty state is derived from draft-vs-saved comparison instead of placeholder flags, which means the page is finally telling the truth about what has changed.

That was the threshold. Persona Studio is no longer just a display surface wearing editor clothes. It is now a real editor.

## Command Center

Command Center needed a different kind of rescue.

Its problem was not just aesthetics. It was that everything arrived at the same visual volume. Status, IDs, event text, health rows, timestamps, approvals, raw detail. It all sat there yelling with the same face. The page technically contained useful information, but it read like operational soup.

So the first pass was hierarchy. I pushed it toward a clearer order: summary, health, detail. That made it more readable, but it still didn't feel like it belonged to the same product as everything else. It still had that bargain-bin diagnostics energy. Useful, maybe. But not integrated.

So I did another pass focused on library convergence. Better shared surface language. Better pill and chip behavior. Better card rhythm. Less of the page feeling like a custom debug artifact that wandered in from a different app.

Then came the semantic pass.

Status rendering was still too locally improvised, so I pulled it onto a canonical frontend runtime token path. Instead of letting Command Center invent its own styling logic for state, I created a shared status registry and resolver. Known values now resolve through a single contract path, and unknown values stay visible without masquerading as something healthier than they are.

That mattered enough that I hardened it with direct tests.

## Contract Hardening

One of the better moments today was catching a false sense of completion.

We had runtime token contract tests, but the frontend Vitest config was not actually discovering `contracts/**` by default. In other words, the tests existed, but the guardrail was still partly imaginary. That got fixed.

It was a small file change, but an important one. The difference between "we wrote the test" and "the test runs by default" is the difference between a ritual and a guardrail.

I also hardened the resolver behavior itself so fallback stays visible, unknown values do not disappear, and prototype-chain nonsense cannot sneak through the front door wearing a forged name tag.

## What Changed Today, Really

The visible output is a better Persona Studio and a more legible Command Center.

But the deeper thing that happened today is that both surfaces got pulled closer to truth.

Persona Studio now behaves like a place where edits actually happen. Command Center now behaves more like a system surface and less like a raw dump of operational leftovers. And underneath both of those, I tightened the semantic layer so repeated meanings are less likely to drift quietly over time.

This was not glamorous work. It was better than glamorous work. It was the kind of work that makes later work safer.
