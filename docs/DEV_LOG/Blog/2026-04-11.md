# Daily Log — April 11, 2026

Today was a contract day more than a feature day. The work moved retrieval from a collection of hidden decisions into a policy the system can name and explain. That sounds small on paper, but it matters because retrieval is where behavior diverges first when a system starts drifting.

The backend change came first. Retrieval overrides now merge into an effective policy instead of replacing anything outright, and the broker carries that policy through the trace. The new diagnostics route can surface a canonical retrieval-posture snapshot, but it is honest about its source: if the backend has not emitted the canonical field yet, the route synthesizes the posture from the existing trace fields. That is the right shape for now. It keeps the boundary explicit without pretending the backend is further along than it is.

Then the UI caught up. The Command Center now has a narrow hook and a dedicated panel for retrieval posture, and it can explain whether a run stayed in conversation, widened within project scope, or crossed into personal knowledge. The explainer text is derived from the same canonical fields, so the operator view is reading the policy rather than guessing at it. On the chat side, the new-thread empty state stopped being generic when a preferred name exists, and the completion payload started carrying identity fields on purpose instead of inheriting them by accident.

The deeper shift is that identity and retrieval are now flowing through the system as explicit inputs rather than ambient context. Preferred name, profession, and guardian name cross the shell, request, task, and prompt boundary. The architecture note made the plane separation legible too: intent, interpretation, policy, execution, observability. That is useful because it gives future changes a place to belong instead of letting them smear across layers.

What remains open is evidence quality. The supported-path proof artifact is back on main, but the current tip still needs fresh live Compose proof, and the new retrieval-posture path is still code/test-level truth rather than runtime proof. Next step is straightforward: emit the canonical posture directly from the backend and then rerun the live proof so the new contract is backed by current evidence.
