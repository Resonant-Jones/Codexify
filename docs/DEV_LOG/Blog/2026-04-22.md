# Daily Log — April 22, 2026

This was a day of subtracting false signals before adding anything new. The visible interface was trimmed first: unfinished beta surfaces came out of the primary nav, the dashboard stopped leaning on demo gallery behavior, Documents lost its nested shell, and Settings was re-centered around the surfaces that are actually meant to stay. That kind of cleanup does not make headlines, but it matters because it narrows the product to the parts we can support honestly.

The backend work followed the same pattern. Tool-loop observability stopped living only in transient worker state and was written into finished-run metadata instead, which means post-run inspection no longer depends on catching a worker in the act. On top of that, the post-completion eval spine landed: durable trace snapshots, a deterministic groundedness check, a best-effort eval worker, and a route for reading the latest eval diagnostics. The system prompt also got a sharper current-turn priority block, which is a small change with real implications for how much older context can pull the conversation off course.

Identity and import handling were tightened at the same time. Verified personal facts now enter the provider-ready context path in a bounded way, with the broker surfacing exactly which fact ids were included and the prompt builder labeling the block as verified. The ChatGPT import path learned to extract third-person model_editable_context facts before that content is filtered away, and it now truncates long fact values so the database column width does not silently kill persistence. Alongside that, the capability-resolution seam was made explicit, and the architecture docs were expanded to pin down Flow Builder threading, continuity governance, and the line between contracts and live behavior.

The last part of the day was the most useful because it refused to flatter the work. The supported-path proof was rerun on the exact current main tip, and it still failed. The runtime is not in the supported local-only posture, upload still dies on missing agent_extension_* tables, and the bounded tool-loop cases still collapse into plain answers. That is not a pleasant result, but it is the right one to carry forward, because the docs now reflect the gap instead of hiding it. It leaves the next pass with a clear job: fix the runtime truth first, then prove it again.
