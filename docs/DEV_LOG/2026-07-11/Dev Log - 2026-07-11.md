Dev Log - 2026-07-11

Today was a boundary day. The work did not expand the product in a flashy way. It made the system harder to misuse. That mattered because the fastest way for a repo like this to lose truth is for access paths, evidence paths, and identity rules to stay implicit while the code around them keeps moving.

The first correction was in Guardian evidence handling. The packet fixture refresh and the `--evidence-packet` drift repair were not about adding capability so much as keeping the checked-in shape honest. The dry-run count diagnostics made the reducer inspection path more legible, which is a small change with real value: the tool now says more about what it is seeing without pretending to be a runtime authority. The fixture freshness audit and its classification corrections served the same goal from the documentation side. They made the record of what changed line up with the actual diff shape.

The second correction was access. The private Cloudflare preview lane, the login and banner adjustments, and the Tailscale tester sidecar all pushed in the same direction. They turned a vague idea of "restricted access" into separate, named surfaces with distinct rules. That is the kind of work that usually only looks administrative from far away. In practice it reduces accidental exposure, narrows operator ambiguity, and makes it possible to reason about who can reach what without guessing which environment is being discussed.

The third thread was identity and participation. The invite lifecycle ADR and the space participant resolution ADR are not runtime proof, but they are still important because they define what the system believes a participant is and how that state should live over time. That kind of model work matters when the rest of the stack starts depending on it. It also makes the architecture index and surrounding docs more useful as a real lookup surface instead of a backlog of ideas.

The day ended up being a mix of contracts, isolation, and archival cleanup. The recipe index, the Tailscale tester recipe, the daily audit, and the dev log batch do not change runtime behavior, but they reduce future friction by making the operator path and the historical path easier to recover. The Peekaboo demo packet sits in the same category: a packaged artifact slice that suggests a more deliberate demo workflow, even if it is not a runtime change.

What changed underneath the surface is straightforward. The repo stopped depending quite as much on context in people's heads. More of the important boundaries now have names, fixtures, tests, or docs attached to them. That is not glamorous work, but it is the kind that makes later work safer.
