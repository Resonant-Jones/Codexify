Narrative Log

Dev Log - 2026-04-25

This was a day about subtraction. Documents had started to carry too much of its own shell logic, and the fix was to pull that responsibility back into `AppShell` so the shared structure actually means something. Once that happened, the Documents view could shrink into the role it should have had all along: a center pane with document interaction, not a mini application frame of its own.

The change was easy to miss at first because it looks like layout work, but the real shift was about authority. `AppShell` now owns the sidebar, the center lane, and the right workspace surface in one contract. `DocumentsView` still decides how documents are listed, uploaded, and opened, but it no longer gets to define the shell around them. That removes a whole class of drift where one route quietly diverges from the rest of the app.

Underneath that, workspace layout mode became more explicit. The drawer is no longer just open or closed; it now carries posture truth that the shell can use to distinguish chat focus, balanced split, and workspace focus, including per-thread persistence. That is the kind of detail that usually disappears into incidental UI code until it starts causing confusion. Pulling it into the shell makes the behavior legible again.

The tests tell the story clearly. They no longer just check that Documents renders; they check that the old route-local rail is gone, that the shared shell anchor exists, and that the drawer still behaves correctly as the viewport changes. That is not glamorous work, but it is the right kind of boring. It makes the system easier to trust.

By the end of the day, Documents was no longer pretending to be its own shell, and the app had a single place where workspace truth lives. That gives the codebase a firmer frame for future routes and makes layout regressions less likely to spread quietly.
