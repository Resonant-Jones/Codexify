Dev Log - 2026-04-15

Summary

Architecture truth-alignment and shell hardening day.

Most of the day was about making the product’s surface grammar and runtime proof line up: Persona Studio was narrowed into an explicit module, diagnostics and identity ownership were separated, command-center retrieval posture became inspectable, and the desktop shell was taught how to carry the runtime assets it needs. The UI polish was not cosmetic in the usual sense; it was the visible cleanup of those ownership lines.

The runtime story mattered as much as the surface story. Earlier in the day, the supported local Compose path was live-proved on `main`; later work hardened the multimodal local path, the composer, and the desktop bundle so the system can keep following that path without depending on hidden assumptions.

What I completed

### Runtime proof and desktop shell
- The supported Compose path was re-proved on `main` earlier in the day, including chat completion, document ingestion/retrieval, and a populated retrieval-posture surface.
- The DMG ritual proof found the packaged app bundle was missing the runtime assets the launcher expects.
- The Tauri build and bundle config were updated to stage backend, guardian, docker, plugins, scripts, requirements, tests, and env templates into the bundle.
- That closes a real packaging gap, but the new staging still needs a fresh end-to-end run on the final head.

### Architecture docs and decision records
- The design-law lane was updated with a module-header / secondary-pill-nav canon, a native presentation SDK contract, a Persona Studio design contract, a surface-class matrix, and a plugin-native shell checklist.
- ADRs were added for imprint ownership and the Flow Builder elicitation lane, which makes the architecture more explicit about where identity lives, where diagnostics belong, and how vague goals become structured work.
- The PI integration spec now frames Pi as an adapter-backed coding agent rather than a second identity or state system.
- This is the day’s clearest truth-alignment work: it turns repeated implementation instincts into written boundaries.

### Persona Studio and surface normalization
- Persona Studio was split into a two-lane shell, then demoted so utility and diagnostics surfaces no longer crowd the primary configuration flow.
- The editor was flattened into a configuration rail, and a later fix restored section nesting after that refactor.
- Dashboard and Documents were normalized alongside it so the app’s main surfaces follow the same rhythm instead of drifting into one-off layout rules.
- The net effect is less ceremony around the actual work and fewer ambiguous surface roles.

### Command Center and retrieval posture
- The Command Center now supports pinned retrieval posture, history pinning, comparison between pinned and current posture, and a diff-note action.
- The trace workbench is no longer just a trace dump; it now shows posture as a user-facing object that can be compared and reasoned about.
- That matters because retrieval policy is starting to behave like a control plane, not an internal detail hidden behind logs.

### Chat composer, mobile rails, and multimodal routing
- The composer regained its transparent input, alignment, send icon, and margin rhythm.
- The attach-file action now drives the real file input, user bubbles are left-aligned with padding, and mobile send-state handling is less brittle.
- Gallery interaction feedback, mobile nav rail selection/momentum, and settings density were tightened so the shell feels more coherent across surfaces.
- Local multimodal routing was also hardened: gemma models are now recognized as vision-capable locally, Ollama-native image payloads are transformed correctly, and request-built instrumentation fires before dispatch.

### Operational tooling and ingest
- The repo gained log-processing and log-refinement scripts, which gives the daily record a repeatable pipeline instead of a one-off manual pass.
- A `docker compose` `obsidian-ingest` profile was added to mount a vault read-only and run `guardian.cli.ingest_cli ingest-obsidian /vault --rebuild` against the shared local vector store.
- The supporting docs around module grammar, design-law usage, and plugin-native shells were updated at the same time, so the tooling and the doctrine moved together.

Important outcome

Persona Studio is no longer just a page with controls. It now has a defensible module grammar, a narrower remit, and clearer neighbors. Diagnostics, identity governance, and authored persona composition are getting pulled apart instead of being stacked on top of each other.

The runtime boundary moved too. The system now has live proof that the supported local path works, and the desktop shell is being pushed toward a bundle that actually carries the runtime it needs. That is not flashy work, but it removes a class of failure where the product looks packaged while still depending on the source tree at launch time.

Narrative Log - 2026-04-15

Today was mostly a day of removing ambiguity. The biggest work was not adding flashy new capability so much as deciding what each surface is actually responsible for and then making the code, docs, and proof artifacts agree with that decision. That showed up in Persona Studio first, where the editor was pushed into a clearer two-lane shell, utility and diagnostics were demoted, and the section nesting was cleaned back up after the refactor. The point was not visual novelty. It was to stop Persona Studio from pretending to own things it does not own.

The same discipline showed up in the architecture docs. The design-law lane got tightened with the module-header and secondary-pill-nav canon, the native presentation SDK contract, the surface-class matrix, and the plugin-native shell checklist. Persona Studio then got its own design contract, while the ADRs pulled identity ownership and Flow Builder elicitation into explicit policy. Dashboard and Documents were normalized along the same axis so the app’s main surfaces follow the same rhythm instead of each inventing its own layout grammar. That is the boring-but-important kind of work that keeps the product from drifting: the system becomes more legible because the boundaries are spelled out instead of implied.

There was also a useful collision between proof and packaging. Earlier in the day, the supported local Compose path was live-proved on `main`, including chat completion and retrieval posture. Later, the DMG ritual showed the opposite kind of truth: the desktop bundle still did not contain the runtime assets the launcher expected. The Tauri build was then taught to stage those assets explicitly. In other words, the launcher stopped relying on the source tree being present and started moving toward a real packaged boundary.

The rest of the day filled in the gaps around that same structural shift. The Command Center went from trace viewer to retrieval-posture surface, with pinned posture, history, and diff notes making the policy legible to a human. The chat composer recovered its alignment, transparent input, attachment path, and send-state handling. Mobile rails, gallery feedback, settings density, and user bubble alignment were all tightened so the shell felt like one system instead of a stack of unrelated layouts. Local vision routing also got corrected: gemma models now route through the Ollama-native image path, which matters because attachment and multimodal behavior only count if the runtime can actually carry them.

The day ended with a small but meaningful tooling pass. The repo now has a log-processing and log-refinement pipeline, plus a vault ingest profile in compose. That is not the same class of work as runtime hardening, but it belongs to the same project of making the system more truthful and more maintainable. The next step is straightforward: re-prove the final head, confirm the packaged desktop assets, and reconcile the short-horizon state doc so the written truth stops lagging the live one.
