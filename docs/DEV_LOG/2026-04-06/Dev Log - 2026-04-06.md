**Dev Log - 2026-04-06**

**Summary**

runtime visibility and boundary-hardening day

The day was about making runtime truth explicit across executors, thread motion, and asset lineage. The biggest shift was not feature count, it was that the system now exposes more of what it knows about itself and carries that truth through the API, UI, and runtime container boundaries.

The other important thread was operational. I added CLI support into the runtime containers and recorded live proof that core chat stays usable while import backlog and background worker failures are present. That proof is partial, but it is still better than assumption.

**What I completed**

**Executor contracts and health**

- Added the canonical executor registry contract and bound delegation worker resolution to it, so executor selection is no longer a loose string lookup.
- Added ExecutorAvailabilityState and ExecutorAuthState, plus /api/health/executors, which reports install state, resolved binary path, auth state, availability, and supported auth modes.
- Added test coverage for the new protocol tokens, executor health model, and health route.

**Runtime container support**

- Added Codex and Claude CLI support to the backend and worker runtime images, with a shared HOME volume so supervised delegation can persist local credential state across restarts.
- This is a prerequisite for server-side execution, not proof that the full delegation loop is complete.

**Thread motion and sidebar recency**

- Added thread_moves and last_interaction_at to the chat schema, and surfaced project names and recency through thread payloads.
- Added explicit thread move and thread snapshot routes, replacing implicit project_id patching with an auditable move action.
- Sidebar ordering now follows last_interaction_at, so the list tracks actual use rather than creation order.
- AppShell now passes the effective documents project id into the workspace drawer, so the shelf and composer can stay aligned with the active project.

**Composer and workspace asset flow**

- Let workspace shelf items be dragged into the composer as structured assets, with cross-context checks so they only attach when the project or thread scope matches.
- Thread and project identity now flow into the composer, which lets the empty-state copy and drop guard reflect the active context instead of guessing.
- Kept the composer affordance tighter after the rework with a smaller send button.

**Runtime proof**

- Added a live proof note showing that thread creation, message persistence, completion requests, and health endpoints stay usable during import backlog.
- The proof also confirmed Postgres-first durability and that Neo4j or embedding-worker failures do not block the core chat path.
- It surfaced a real schema gap around thread_moves and the existing Chroma dimension mismatch, so the proof should be read as partial runtime truth, not release signoff.

**Important outcome**

Executors are no longer just names in code. They now have a registry contract, a health surface, and explicit availability and auth states that can be inspected at runtime. That matters because delegation moves from being an assumption to being something the system can actually report on.

Thread movement also stopped being a hidden side effect. The move endpoint, audit table, and last_interaction_at ordering make the sidebar and project history line up with what users actually did, not what the database happened to store first. The system is starting to reflect motion instead of reconstructing it afterward.

The composer changed in the same direction. Workspace assets can now enter as structured attachments, but only when they belong to the right project or thread, which keeps the convenience path from erasing context boundaries. That is the kind of change that makes a UI more trustworthy without making it noisier.

**Narrative Log**

Dev Log - 2026-04-06

Today was a day of turning hidden state into explicit state. The executor side now has a canonical registry, health modeling, and a live endpoint that says whether a binary is installed, what auth state it can infer, and how the runtime should classify it. The container changes mattered for the same reason: Codex and Claude now have a place to live inside the runtime, with a shared home volume so supervised delegation can persist across restarts instead of depending on ephemeral container state.

The chat boundary work followed the same pattern. Thread moves stopped being an implicit project_id patch and became a first-class move action with an audit row and a dedicated snapshot route. last_interaction_at now drives sidebar ordering, so the list reflects actual use instead of creation order, and the thread payload carries enough project context for the UI to label things without guessing. That is not flashy work, but it removes ambiguity where the interface used to reconstruct it.

The composer moved in the same direction. Workspace shelf items can now be dragged into the composer as structured assets, but only when they belong to the same project or thread scope. That guard matters because the new convenience path should not quietly erase ownership boundaries. The send button cleanup was minor beside that, but it fits the same theme: after the structural change, the affordance itself got tighter and clearer.

The strongest evidence came from the live import proof. Core chat still works while import backlog exists, health endpoints stay responsive, and background failures in graph enrichment or embedding do not block the user-facing path. That is a real operational result. It also surfaced the remaining seams cleanly, especially the thread_moves schema gap that was present in the proof window and the separate Chroma dimension mismatch that still needs a follow-up. The system is more honest now, but not finished. The next step is to fold today’s truth back into the short-horizon state and keep pushing the delegation path toward something that can actually be proven end-to-end.
