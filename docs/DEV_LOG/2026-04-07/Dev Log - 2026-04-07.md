  
Dev Log - 2026-04-07

Summary

Audit hardening and runtime truth day.

Today was mostly about forcing the system to say what it is actually doing. A large part of the work built a repeatable regression-audit layer so risk, gates, and manual attestations are explicit instead of implied. The other part hardened the message path so attachments, document context, and image payloads survive the trip through chat and routing without being flattened into generic text.

The day mattered because it reduced ambiguity in two places that govern trust: release confidence and runtime context handling. The result is a more legible system for operators and a more faithful set of inputs for the model.

What I completed

Regression audit stack

- Added audit tooling, templates, and generated gate/risk snapshots.
- Made the first gate pass mechanically honest so deterministic checks are separated from manual attestations.
- The deterministic gate snapshot is green, but blast radius, rollback path, identity isolation, golden tasks, cross-provider checks, schema migration confidence, retrieval evals, error-rate monitoring, and cost anomaly review still require human judgment.

Chat and context handling

- Refactored document interactions for scoped thread tiles so workspace previews, composer state, and chat context stay aligned.
- Added structured document-context markers and parsing so attached documents and in-message excerpts survive serialization.
- Tightened image attachment handling so multimodal providers receive real image payloads and text-only providers get derived context instead of a broken multimodal shape.

Operator surfaces and docs

- Added sidebar source logos so thread provenance is visible at a glance.
- Added setup instructions and builtin help docs so the first-run path is explicit instead of tribal knowledge.

Runtime and storage hardening

- Added vector doc delete support and included bubblewrap in the backend runtime image.
- Captured a live concurrent import catch-up proof showing the import worker could drain backlog while normal chat stayed usable.

Important outcome

The system is no longer just carrying attachments through chat and hoping the model infers intent. It now preserves document and image context as structured state, routes image payloads according to provider capability, and keeps the UI aligned with that shape.

The system stopped treating risk as commentary. It now emits explicit gate and risk artifacts, and the live import proof showed that background catch-up can run without taking chat offline. That is a narrower, more honest failure mode.

Narrative Log

Today was a correction day. The project spent a lot of time making hidden assumptions visible: what counts as a gate, what counts as risk, what counts as an attachment, and what the runtime is allowed to do while backlog work is still in flight. None of that is flashy, but it is the difference between a system that feels plausible and a system that can be trusted.

The first shift was on the audit side. The new regression tooling, templates, and generated snapshots turn change review into something repeatable. The mechanically honest follow-up matters just as much as the tooling itself: deterministic checks are now separated from manual judgment instead of being blurred together. That makes the release surface more truthful, even though it still leaves the human attestations in place.

The second shift was on the context path. Document markers now survive as structured context instead of disappearing into plain text, and image payloads now follow the correct route depending on whether the provider can actually accept multimodal input. That flows all the way through thread tiles, workspace previews, and the composer, so the UI and the backend are describing the same thing instead of competing interpretations of the same message.

The live proof at the end was the real test. An import worker drained a real backlog while chat stayed usable, health surfaces stayed up, and the assistant response persisted normally. The system was under pressure, and the proof is honest about the cost, but it showed that background catch-up can coexist with normal use instead of forcing a shutdown or a guess.

The smaller changes fit the same pattern. Sidebar logos made source provenance visible. Setup and builtin help docs made the first-run path less dependent on memory. Vector delete support and bubblewrap tightened the runtime. Taken together, the day moved Codexify toward a system that is easier to operate because it is harder to misread.

That makes the next steps clearer: refresh the short-horizon state, finish the remaining manual gate attestations, and keep pressure-testing attachment and context behavior until it is boring.
