# Reference Docs

`docs/reference/` is the canonical documentation surface for Codexify.

## Sections

- `architecture/`: system maps, flows, storage, config, and risks
- `product/`: platform overviews, install/config guides, policy docs, and high-level product material
- `guardian/`: subsystem-specific runtime and validation docs
- `infrastructure/`: deployment, sync, and system integrity docs
- `integration/`: external integration notes
- `plugins/`: plugin SDK and plugin-facing docs
- `security/`: security and auth boundary decisions
- `development/`: engineering specs and dev-only implementation references
- `prompts/`: canonical prompt and governance docs
- `operator/`: CLI, help, release, desktop, UI, debug, and operations docs

Use this tree for stable material you expect other docs to link to repeatedly.
