# Paul Report Output Templates

**For:** Paul's agent - standardized shapes for study reports
**Last updated:** 2026-06-29

## General Study Report Template

Use this template for most first-step studies.

- **Report title** - descriptive and specific.
- **Area inspected** - which directory, subsystem, module, or surface.
- **Files/docs read** - specific paths inspected.
- **What I found** - key observations grounded in evidence.
- **What seems important** - why this matters to Codexify.
- **What is confusing or unresolved** - things that remain unclear.
- **What this report does not prove** - explicit limits of the inspection.
- **Questions for Resonant** - things only Resonant can clarify.

## Memory Trace Study Template

Use this template when the study is about memory, identity, provenance, or long-lived context.

- **Memory surface inspected** - what boundary or mechanism was studied.
- **Governing docs and code** - contracts, docs, and files that define it.
- **What persists** - what survives over time.
- **What is derived** - what is rebuilt from other data.
- **What is intentionally forgetful** - what the system does not keep.
- **Where source-of-truth lives** - canonical owner of the state.
- **Where context is reconstructed** - tracing, retrieval, or fallback paths.
- **Risks and open questions** - what still needs constraint or proof.
- **Questions for Resonant** - boundary intent questions.
