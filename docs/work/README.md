# Work Docs

`docs/work/` is the execution ledger for Codexify.

## Sections

- `campaigns/`: campaign definitions, grouped by `YYYY/MM`
- `tasks/`: task artifacts, grouped by `YYYY/MM`, plus `chunks/` and `runner-failures/`
- `plans/`: planning material, roadmaps, and future feature work
- `reports/`: audits, evaluations, and roadmap/status reports
- `investigations/`: focused diagnoses and incident-style investigations

## Conventions

- New campaign docs belong under `docs/work/campaigns/YYYY/MM/`.
- New task docs belong under `docs/work/tasks/YYYY/MM/`.
- Historical execution context stays here even after completion; do not move completed work into `archive/` unless it is duplicate or superseded.
