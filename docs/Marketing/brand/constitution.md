# Resonant Constructs Brand Constitution (Codexify Context)

## Mission Context

Resonant Constructs builds local-first cognitive infrastructure for operators who need systems that stay truthful under real constraints.
Codexify is the flagship product surface for that mission.

## Worldview Commitments

1. Local-first operation is a feature, not a fallback.
2. Identity boundaries are explicit, enforceable, and auditable.
3. Failure states must be visible; silent degradation is unacceptable.
4. Release claims must match observed evidence.
5. Product language should reduce ambiguity, not inflate aspiration.

## Positioning Frame

Codexify is a practical operating layer for local-first AI workflows, with explicit policy surfaces, evidence paths, and bounded execution seams.

## Forbidden Claim Patterns

Do not claim or imply:

- public launch readiness without a dated evidence artifact
- that desktop shell and Docker web runtime are the same supported path
- guaranteed outcomes, infallibility, or autonomous safety
- capabilities not backed by campaign receipts, current-state docs, or proof artifacts
- "set and forget" automation where human supervision remains required

## Required Language Discipline

When uncertainty exists:

- state uncertainty directly
- label the evidence tier (`implemented`, `verified`, `live-proven`)
- separate roadmap intent from current behavior
