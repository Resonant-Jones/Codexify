# Current Truth Claim Boundary

Governing law:

> Core is not everything that exists. Core is what can be supported, proved, and defended as a coherent product today.

Primary claim gate source:
- `docs/architecture/00-current-state.md` (highest priority)

## Safe to Claim

| Theme | Safe claim posture |
|---|---|
| Local-first posture | Codexify is currently supported on a local Docker Compose path with local-first defaults. |
| User-owned knowledge | Codexify emphasizes user-owned, inspectable knowledge workflows. |
| Project/thread continuity | Codexify keeps thread/project continuity as a central workflow surface. |
| Upload + retrieval path | Upload -> embed -> readback is part of supported-path evidence. |
| Runtime honesty | Runtime and health surfaces are explicitly part of operational truth. |
| Inspectable behavior | Operators can inspect system state through health/catalog/events/proof artifacts. |
| Knowledge capture | Conversations and documents can be captured as durable working context. |
| Workflow encoding | Workflow thinking can be encoded through explicit system seams and artifacts. |
| Provenance and claim discipline | Claims are expected to be evidence-linked and tiered. |

## Claim with Caution

| Theme | Caution posture |
|---|---|
| Command Center | Claim only as an operator-facing, non-dispatch/recommendation-oriented control seam. |
| Command bus | Treat as implementation/runtime seam; avoid broad end-user release promises. |
| Desktop shell posture | Do not collapse desktop shell and supported Compose path into one claim. |
| Provider breadth | Do not imply broad cloud-provider support as current default posture. |
| Connector ecosystem | Mention as evolving or constrained; avoid maturity claims. |
| Enterprise deployment language | Use conditional/future framing unless current proof explicitly supports production guarantees. |
| Graph context and graph writes | Keep default-off and scoped; do not market as core release behavior. |
| Federation | Treat as future-facing/limited unless live proof and support posture are explicit. |

## Do Not Claim Yet

| Theme | Why blocked |
|---|---|
| Fully autonomous agents | Not a current supported release promise. |
| Set-and-forget automation | Violates supervision/truth-surface posture. |
| Mature hosted SaaS product | Conflicts with current local-first supported path framing. |
| Broad enterprise-ready guarantee | Exceeds current release-proof scope. |
| Guaranteed model/provider reliability | Runtime variance and queue/provider realities remain explicit. |
