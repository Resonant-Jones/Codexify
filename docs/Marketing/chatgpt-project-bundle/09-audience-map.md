# Audience Map

## Segment Matrix

| Segment | Pain | Desired transformation | Relevant Codexify promise | Claim risks | Best content angles |
|---|---|---|---|---|---|
| Solo AI builders | Tool sprawl and context loss | One durable operating surface | Local-first continuity and inspectable state | Overpromising automation autonomy | Build-in-public workflows, setup walkthroughs |
| Neurodivergent/systems thinkers | Cognitive overload from fragmented contexts | Externalized structure with low-friction recall | Workspace continuity with explicit boundaries | Overstating cognitive-health outcomes | Ritual guides, continuity templates, calm systems literacy |
| Researchers and writers | Source drift and weak retrieval recall | Reliable research memory layer | Documents + threads become retrievable context | Overclaiming citation/completeness guarantees | Research pipeline posts, provenance explainers |
| Local-first/privacy-conscious users | Dependence on centralized AI stacks | Owned infrastructure and data posture | Local-first supported path with user control | Implying absolute privacy/security guarantees | Local deployment stories, threat-model explainers |
| Small teams operationalizing AI | Knowledge siloing and inconsistent process | Shared project intelligence continuity | Institutional memory and claim discipline | Overclaiming enterprise readiness | Team playbooks, weekly ritual kits |
| Consultants installing AI workflows | Hard-to-replicate client systems | Repeatable implementation templates | Workflow encoding and truth-surface coaching | Overclaiming turnkey connector maturity | Client implementation case studies, operator checklists |

## Segment Prioritization Guidance

- Primary now: solo builders, local-first/privacy-conscious users, researchers/writers.
- Secondary now: small teams and consultants (with conservative claims).
- Use explicit caution tags for enterprise-adjacent language.
