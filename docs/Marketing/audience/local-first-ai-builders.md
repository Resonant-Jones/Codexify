# Audience Pack: Local-First AI Builders

## Primary Audience

Solo operators and small technical teams building AI systems that must run reliably with local control, explicit permissions, and clear runtime boundaries.

## Core Needs

- trustable runtime truth over hype
- bounded execution and inspectable behavior
- identity-aware policy seams
- operational resilience under partial failure
- evidence-backed release posture

## Pain Signals

- "The tool says it works, but I cannot prove it."
- "State drifts across environments and nobody can explain why."
- "Permission boundaries are implied instead of enforced."
- "Marketing says one thing, runtime reality says another."

## Message Fit

Codexify should be presented as dependable infrastructure for serious builders:

- explicit contracts
- visible failure surfaces
- evidence-linked claims
- local-first operator control

## Channel Tone Defaults

- Website: precise and structural
- Social: concise and evidence-first
- Community: practical and conversational, with implementation honesty
