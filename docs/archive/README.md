# Archive Docs

`docs/archive/` keeps documentation that still has context value but is no longer canonical.

## What Belongs Here

- prompt workbench notes and mirrors
- legacy task artifacts with non-canonical names
- duplicate campaign docs retained for provenance
- scratch or superseded material that should not be the primary target of new links

Archive docs are preserved, but canonical references should prefer `docs/reference/` or `docs/work/`.
