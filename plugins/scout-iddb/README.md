# Scout-IDDB Plugin

## Purpose

The Scout-IDDB plugin provides a **dormant scaffold** for mapping Scout logs into semantic nodes within the Identity-Driven Database (IDDB). This plugin defines the schema, types, and processing pipeline needed to transform observability data into graph-structured knowledge.

## Current Status: DORMANT

This plugin is currently **inactive and non-interfering**:

- ✅ Schema definitions and type structures are provided
- ✅ Stub runtime functions are available
- ❌ **No IDDB writes occur**
- ❌ **No automatic activation**
- ❌ **No imports into Codexify core modules**
- ❌ **No side effects on runtime behavior**

The plugin must be **manually enabled** and integrated before it performs any actual operations.

## Architecture Overview

### Node Types

The plugin defines the following IDDB node types for representing Scout data:

1. **ScoutLog**
   - Primary log entry from Scout
   - Contains: timestamp, severity, message, tags, metadata
   - Serves as the central node for observability events

2. **Component**
   - Represents a system component or service
   - Contains: name, type, version
   - Connected to logs that describe component behavior

3. **URL**
   - Represents a URL or endpoint being observed
   - Contains: full URL, domain, path, query parameters
   - Connected to logs that observed the URL

4. **RuntimeInstance**
   - Represents a specific runtime or execution context
   - Contains: instance ID, environment, version
   - Connected to logs generated by that instance

### Relationship Types

The plugin defines the following relationship types:

1. **OBSERVED_AT**
   - `ScoutLog -> URL`
   - Links a log entry to the URL it observed

2. **DESCRIBES**
   - `ScoutLog -> Component`
   - Links a log entry to the component it describes

3. **GENERATED_BY**
   - `ScoutLog -> RuntimeInstance`
   - Links a log entry to the runtime that generated it

All relationships are currently annotated with `"state": "pending"` to indicate they are not actively being created.

- **No side effects** in any module exports
